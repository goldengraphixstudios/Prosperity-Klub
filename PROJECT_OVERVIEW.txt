Prosperity Klub Project Overview
Generated by Codex after reviewing the repository.

1. What this website is

Prosperity Klub is a Next.js website for a Filipino financial education, guidance, and wealth-building community connected to International Marketing Group (IMG). The core positioning is "guided, not sold": the site presents Prosperity Klub as the education, mentorship, and community layer, while IMG and other partner institutions provide financial infrastructure, provider access, and execution pathways.

The site is aimed at Filipinos and OFWs who want better financial clarity, protection, savings discipline, investment direction, debt strategy, membership support, and long-term income growth opportunities.

2. Technology stack

- Next.js 16 App Router
- React 19
- TypeScript
- Tailwind CSS v4
- Framer Motion
- Lenis smooth scrolling
- Radix/shadcn-style UI primitives
- Supabase server client for live data capture and admin reads
- GitHub Pages workflow for static export, although server API routes mean a normal Next.js host is a better deployment fit for all features

Home page:
- Introduces Prosperity Klub as a financial education and growth community.
- Highlights "Guided, Not Sold" and IMG infrastructure.
- Promotes free financial clarity sessions, membership, the free ebook, success stories, FAQs, and the three-step journey: Get Clarity, Get Matched, Grow With Community.

- Emphasizes mentorship, accountability, financial protection, wealth habits, income growth, and long-term community building.

How It Works page:
- Explains the customer journey from first click to conversation, qualification form, guidance and matching, then follow-up and growth.
- Reinforces that the first interaction is for clarity, not pressure.
- Explains why the qualification form matters and where IMG fits.

Resources page:
- Acts as a Financial Learning Center.
- Promotes a free ebook titled "The Secret to Saving and Building Your Future".
- Shows only an ebook cover preview on-site.
- Uses a gated request dialog asking for name and email.
- Submits ebook requests to the backend route and Supabase table.
- Links to resources, tools, partners, and membership.

Book page:
- Embeds a Calendly booking iframe for a free financial clarity session.
- Explains expectations for the session.
- Provides WhatsApp and email alternatives.

Contact page:
- Contains the main lead form.
- Collects full name, email, phone, location, OFW status, income range, and financial priorities.
- Submits to the backend leads API, then redirects to the thank-you page.

Membership page:
- Presents two membership paths:
  - Full Access Membership
  - Ipon Challenge installment option

- Submissions are sent to separate Supabase-backed API routes.

Membership Submissions page:
- A transitional/admin-note page explaining that membership submissions are no longer stored in browser storage.

Links page:
- Provides organized external links for:
  - Partner institutions

Partners page:
- Explains the trusted-institution ecosystem.

Savings Calculator page:
- Interactive calculator for savings planning.
- Supports three modes:
  - Find required starting balance
  - Find required regular contribution
  - Find time needed to reach a target
- Inputs include target amount, starting balance, regular contribution, frequency, annual interest rate, years, and deposit timing.
- Produces final balance summary, total deposits, interest earned, pie chart, stacked chart, and yearly breakdown table.

Debt Strategies Calculator page:
- Interactive Philippines-focused debt calculator.
- Users can add debts with name, balance, APR, and minimum payment.
- Supports sample data, save/load to localStorage, clearing, calculation, and CSV exports.
- Compares:
  - Debt snowball
  - Debt avalanche
  - Secured consolidation loan
- Consolidation includes APR, term months, origination fee, fixed fees, financed/upfront fees, collateral type, appraised value, and max LTV.
- Outputs payoff time, total interest/cost, balance-over-time charts, payoff order, amortization tables, and eligibility notes.
- The calculations are educational and do not replace lender terms or regulated financial advice.

Admin page:
- Browser UI for a live submissions dashboard.
- Login requires backend admin credentials from environment variables.
- Shows counts and latest records for:
  - Leads
  - Ebook requests
  - Full Access registrations
  - Ipon Challenge registrations

Legal/disclosure pages:

Thank-you page:

4. Core user flows

Financial clarity flow:
- User lands on Home, About, How It Works, Resources, Links, Partners, or Contact.
- Main CTA sends them to Book or Contact.
- Book uses Calendly.
- Contact submits a lead into Supabase through /api/leads.

Lead qualification flow:
- Lead form captures priorities, OFW status, income range, source page, and contact details.
- Backend computes tags such as Savings-Lead, Insurance-Interested, HMO-Interested, Investment-Ready, OFW-Lead, High-Income, Resource-Lead, Contact-Lead, Membership-Interested, Multi-Priority, and Exploratory.
- Backend computes qualification and recommended follow-up text.
- Lead is stored in Supabase.

Ebook flow:
- User requests the free ebook from Resources.
- Dialog collects name and email.
- Backend stores request in the ebook_requests Supabase table.
- Current delivery model appears to be manual or future email automation. The repo includes Supabase notes recommending a future Edge Function for signed URL and email delivery.

Membership flow:
- User reviews Full Access or Ipon Challenge.
- User fills the relevant registration form.
- Frontend validates required fields and consent.
- Backend validates again and stores the record in either full_access_registrations or ipon_challenge_registrations.
- Admin can later review submissions in the dashboard.

Calculator flow:
- Savings Calculator and Debt Strategies Calculator are client-side educational tools.
- They do not require Supabase.
- Debt calculator also supports localStorage save/load and CSV exports.

Admin CRM flow:
- Admin logs in with environment-based credentials.
- A signed HTTP-only cookie is created.
- Dashboard route checks the cookie or an x-admin-key header.
- Dashboard fetches Supabase counts and latest records.

5. Backend/API routes

/api/leads:
- POST endpoint for general lead capture.
- Requires name and email.
- Computes lead tags, qualification status, and follow-up recommendation.
- Stores into Supabase leads table.

/api/ebook-requests:
- POST endpoint for ebook requests.
- Requires name and email.
- Stores into Supabase ebook_requests table.

/api/membership/full-access:
- POST endpoint for Full Access membership applications.
- Validates required fields and consent.
- Stores into Supabase full_access_registrations table.

/api/membership/ipon-challenge:
- POST endpoint for Ipon Challenge registrations.
- Validates required fields and consent.
- Stores into Supabase ipon_challenge_registrations table.

/api/admin/login:
- Validates username/password against ADMIN_USERNAME and ADMIN_PASSWORD.
- Creates a signed admin session cookie.

/api/admin/logout:
- Clears the admin session cookie.

/api/admin/dashboard:
- Requires valid admin session cookie or x-admin-key matching ADMIN_DASHBOARD_KEY.
- Reads Supabase counts and latest records from leads, ebook_requests, full_access_registrations, and ipon_challenge_registrations.

6. Data and database setup

Supabase SQL files define:
- leads
- ebook_requests
- full_access_registrations
- ipon_challenge_registrations

The SQL adds indexes, enables row level security, and allows authenticated reads/updates for admin-style access. Ebook requests include an anonymous insert policy in the standalone ebook setup SQL, but live_crm_setup.sql drops the anon insert policy and only keeps authenticated read/update policies. In the current app code, inserts are done server-side with the Supabase service role key, so anonymous insert policies are not required for these API routes.

Important environment variables:
- SUPABASE_URL or NEXT_PUBLIC_SUPABASE_URL
- SUPABASE_SERVICE_ROLE_KEY
- NEXT_PUBLIC_SUPABASE_URL
- NEXT_PUBLIC_SUPABASE_ANON_KEY
- ADMIN_USERNAME
- ADMIN_PASSWORD
- ADMIN_SESSION_SECRET or ADMIN_DASHBOARD_KEY
- ADMIN_DASHBOARD_KEY

7. Deployment notes

The repo includes a GitHub Pages workflow that runs npm ci and npm run build, then deploys the static out folder.

However, the project contains live Next.js server routes under src/app/api. Those routes are required for leads, ebook requests, membership submissions, and the admin dashboard. GitHub Pages cannot run those server routes. For the full website to work, it should be deployed to a normal Next.js-capable host such as Vercel or another Node/Next platform.

The Next config conditionally enables static export only when GITHUB_ACTIONS is true. It also sets basePath and assetPrefix to /Prosperity-Klub for GitHub Pages.

10. Current caveats and things to know before editing

- node_modules is not present in the checked-out repo. Dependencies need npm install before local development.
- Server routes require Supabase and admin environment variables to work.
- Full production functionality requires a Next.js server host, not only GitHub Pages.
- The ebook request page currently stores requests but does not itself send the full ebook automatically.
- Debt calculator stores saved calculator state in the user's browser localStorage, not the database.
- data/leads.json exists but is currently an empty array and does not appear to be the active storage path anymore.
- AGENTS.md says previous work focused on /debt-strategies and mentions codex-conversation-log.txt, but that log file is ignored and not present in the current tracked file list.
